# Contributing

## Development Workflow

1. Install dependencies.
2. Create a feature branch.
3. Keep changes focused.
4. Run lint, typecheck, tests, and build.
5. Open a pull request with screenshots for frontend changes.

## Commands

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

## Commit Style

Use concise conventional commits:

- `feat(api): add order status workflow`
- `feat(web): add customer checkout page`
- `fix(api): validate cart quantities`
- `ci: add backend test workflow`
- `docs: update deployment guide`

## Pull Request Checklist

- The change is scoped and easy to review.
- User-facing behavior is described.
- Environment variables are documented.
- Tests or verification notes are included.
- CI passes.
